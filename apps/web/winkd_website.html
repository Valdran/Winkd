Security & Privacy
🔒 Signal Protocol — End-to-End Encryption
Every message is encrypted on your device before it leaves. Winkd uses the Signal Protocol — the gold standard in private messaging, the same cryptographic foundation as Signal itself. The server relays encrypted blobs and is mathematically unable to read them. Not even us.
🔁 X3DH Key Exchange
When you add a contact, your client performs an Extended Triple Diffie-Hellman (X3DH) key agreement entirely on your device. Three DH computations across identity, signed pre-key, and one-time pre-key establish a shared secret the server never touches. One-time pre-keys are consumed once and destroyed — no replay possible.
⚙️ Double Ratchet — Forward Secrecy & Break-in Recovery
After key exchange, every message rotates the encryption key via the Double Ratchet algorithm. Compromising today's keys cannot decrypt yesterday's messages (forward secrecy). And after a compromise window closes, the ratchet automatically recovers — future messages return to full protection (break-in recovery).
🔑 AES-256-GCM Message Encryption
Each individual message is encrypted with AES-256-GCM using a fresh random IV. Authenticated encryption means any tampering with a message in transit is detected and rejected before delivery.
🗝 Keys Live on Your Device Only
Your identity key, signed pre-key, one-time pre-keys, and all ratchet session state are stored in your browser's IndexedDB. They are never transmitted to the server. A server breach, a court order, or a rogue employee cannot produce your keys — because we simply do not have them.
🔢 Safety Numbers — Verify Your Contacts
Every conversation has a 60-digit safety number derived from both users' identity keys via SHA-512. Compare it with your contact in person or over a phone call to confirm no man-in-the-middle is present. The same verification method used by Signal.
🔐 Two-Factor Authentication (TOTP)
Enable TOTP 2FA from the Security panel — works with Google Authenticator, Authy, Bitwarden, and any RFC 6238-compatible app. When 2FA is on, correct password alone is not enough: login issues a short-lived challenge token, and a full session is only created after you supply a valid 6-digit code. No session is ever created by password alone.
🗝 Backup Codes — Never Locked Out
When you enable 2FA, Winkd generates 10 cryptographically random backup codes. Each is 128 bits of entropy, hashed with SHA-256 before storage — the plaintext is shown once and never persisted. Codes are single-use. You can regenerate the full set at any time with a valid TOTP code. The Security panel shows how many remain so you're never caught off guard.
🛡 Argon2id Password Hashing
Passwords are hashed with Argon2id at 128 MB memory cost, 4 iterations — one of the strongest password hashing configurations available. A plaintext password never touches the database. Minimum 12 characters with complexity requirements enforced at registration.
🎲 256-bit Cryptographic Session Tokens
Session tokens are 32 bytes of cryptographically random data — not UUIDs, not sequential IDs, not guessable. Tokens are never placed in URLs (which appear in server logs, browser history, and Referer headers). They expire after 30 days.
🚦 Rate Limiting & Brute-Force Protection
Login is limited to 10 attempts per minute per IP. Registration is capped at 5 per minute per IP. Password brute-forcing is blocked at the network level before it can reach the database.
🔐 OAuth with PKCE & CSRF Protection
Sign in with Discord, Google, Apple, and 9 other providers. Every OAuth flow uses PKCE (SHA-256 code challenge) to prevent authorization code interception, and a signed HttpOnly SameSite=Strict cookie to block cross-site request forgery — no server-side state session required.
🔓 WebSocket Auth — No Tokens in URLs
WebSocket connections authenticate via the first message payload, never a query parameter. URL-based tokens appear in server access logs and browser history. Winkd's approach keeps your session token out of any log file. On invalid auth, the server closes with code 4001 — no retry, no session leak.
💻 Multi-Device — Per-Device Key Bundles
Each device you use registers its own Signal Protocol pre-key bundle. Your encryption keys are never shared between devices — if one device is compromised or lost, you revoke it from the Security panel. Revoking a device deletes its pre-key bundle from the server; future messages cannot be initiated to that device.
📋 Security Audit Log
Every security event on your account is recorded: logins (success and failure), 2FA challenges, backup code use, device registrations and revocations. View your last 100 events in the Security panel, including the IP address and timestamp of each event. You'll know immediately if something unexpected happened to your account.
🚫 Zero Data Monetisation
Winkd is open source (MIT licence) with no advertising, no analytics without explicit opt-in, and no data sold — ever. The business model is hosted instances and optional cosmetics. We have zero financial incentive to collect your conversation data.
🌐 Self-Hostable — Full Data Sovereignty
Run your own Winkd server with the provided Docker Compose setup. You own the database, the keys, the connection list. No dependency on Winkd infrastructure. Full source code at github.com/valdran/winkd.
What the server can and cannot see
Cannot see: message content, encryption keys, safety numbers, your password, your TOTP secret, your backup codes (hashed only).
Can see: who you've added as contacts, when you were online, your registered devices, your IP address (standard for any networked service), and your own security audit log events (which you can also see).

A server breach, a subpoena, or a rogue employee cannot access the content of your messages.

About Winkd
Winkd is a free, open-source (MIT) instant messenger built with privacy at its core. No advertisements. No user data sold. Core messaging is free forever. Just messaging with personality.
Built with Tauri (desktop), React Native (mobile), and a Rust WebSocket server. Self-hostable via Docker Compose. Inspired by the design language of MSN Messenger, not a clone, not affiliated with Microsoft.
